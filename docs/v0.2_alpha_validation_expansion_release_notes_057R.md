# v0.2 Alpha Validation Expansion — Release Notes

> Date: 2026-06-06
> Milestone: v0.2 Alpha — validation expansion release-ready
> Status: GO (per release readiness audit 057P)

## 1. New Capabilities

### IS/OOS Stability Gates
- OOS metrics flow through validation pipeline into elimination engine.
- Three stability ratio rules: PF degradation, drawdown ratio, average trade degradation.
- Configurable from `EliminationConfig`; default off.
- Displayed in validation summary widget and exported reports.

### Remove-Best-N Trades Stress Test
- Engine function `stress_remove_best_n_trades()` — removes top N trades by PnL, recomputes metrics.
- Pipeline opt-in via `PipelineConfig.run_remove_best_n_trades_stress`.
- Validate page UI controls (checkbox + N spinbox + threshold spinbox).
- Detail sub-lines in widget, Markdown, and HTML reports.

### Bootstrap Monte Carlo
- `run_bootstrap_monte_carlo()` — trade resampling with replacement, 200 iterations default.
- 95% confidence intervals for PnL, profit factor, and max drawdown.
- Pipeline opt-in via `PipelineConfig.run_bootstrap_monte_carlo`.
- Bootstrap MC card in widget + lines in Markdown/HTML reports.
- Validate page UI controls (checkbox + iterations + confidence).

### Walk-Forward Per-Window Equity
- `WalkForwardWindow.equity_curve` stores per-window equity as `list[float]`.
- Opt-in via `PipelineConfig.wf_store_equity` (default off).
- Widget WF Equity Summary (up to 5 windows).
- Markdown and HTML report tables.

### Opt-in IS Baseline Quality Precheck
- `PipelineConfig.run_is_baseline_quality_precheck` — skips stress/MC/WF when baseline has zero trades.
- `fail_is_baseline_on_nonpositive_pnl` for negative PnL strategies.
- Precheck failure card in widget + lines in reports.
